Spring Batch를 활용한 데이터 처리 및 통계 관리 시스템

🔍 문제 상황

대용량 데이터 처리의 필요성

KernelEngine 서비스에서는 다음과 같은 데이터 처리 작업들이 필요했습니다:

  1. ** 블로그 게시글 수집**: 여러 블로그의 게시글을 일괄 처리
  2. 사용자 통계 생성: 일간 가입자 수, 인기 게시글 등 서비스 지표 계산
  3. 데이터 정제: 수집된 데이터의 중복 제거

기존 방식의 한계

  • 실시간 처리 부담: API 서버에서 대용량 데이터 처리 시 성능 저하
  • 메모리 부족: 모든 데이터를 메모리에 로드하여 처리하는 방식의 한계
  • 트랜잭션 관리: 데이터 처리 중 오류 발생 시 전체 롤백 문제

이러한 작업들은 실시간 API 처리와는 다른 특성을 가지므로 별도의 배치 처리 시스템이 필요했습니다.

🛠️ 기술적 해결 방안

Spring Batch 아키텍처

Spring Batch 프레임워크를 활용하여 견고한 배치 처리 시스템을 구축했습니다:

1. Job과 Step 분리

각 데이터 처리 작업을 독립적인 Job으로 구성하여 관리 용이성을 확보했습니다.

2. 청크 기반 처리

메모리 효율성을 위한 청크 단위 데이터 처리로 대용량 데이터 안정적 처리가 가능합니다.

3. 트랜잭션 관리

배치 작업의 안정성을 위한 트랜잭션 처리로 데이터 일관성을 보장합니다.

배치 작업 설계 및 구현 전략

1. 크롤링 배치 작업

RSS 크롤링 자동화를 위한 배치 시스템 구현:

Job-Step 구조 설계:

  • 크롤링 Job은 블로그 크롤링, 새 게시글 처리, 통계 업데이트 Step으로 구성
  • 각 Step은 독립적으로 실행되며 이전 Step 실패 시에도 재시작 가능
  • RunIdIncrementer를 통한 중복 실행 방지

청크 기반 처리:

  • 10개 블로그 단위로 청크 처리하여 메모리 효율성 확보
  • Reader-Processor-Writer 패턴으로 명확한 책임 분리
  • FaultTolerant 설정으로 일부 블로그 크롤링 실패가 전체에 영향을 주지 않도록 설계

오류 처리 전략:

  • IOException에 대한 3회 재시도 정책
  • CrawlingException 발생 시 5개까지 Skip 허용
  • 성공 시 에러 카운트 리셋, 실패 시 증가로 블로그 상태 관리

2. 통계 집계 배치 작업

서비스 지표 생성을 위한 통계 처리 시스템:

일일 통계 Job 구성:

  • 사용자 통계, 게시글 통계, 인기 태그 계산 Step의 순차 실행
  • 각 Step은 전날 데이터를 기준으로 통계 계산
  • Tasklet 방식과 Chunk 방식을 적절히 조합하여 활용

통계 계산 로직:

  • 신규 가입자 수, 활성 사용자 수 등 사용자 관련 지표
  • 게시글 조회수, 좋아요 수 등 콘텐츠 관련 지표
  • 인기 태그 및 카테고리 분석을 통한 트렌드 파악

3. 데이터 정제 배치 작업

데이터 품질 관리를 위한 정제 시스템:

중복 제거 전략:

  • SQL 쿼리를 통한 중복 게시글 탐지
  • JdbcCursorItemReader로 메모리 효율적인 대용량 데이터 처리
  • 컨텐츠 길이 비교를 통한 더 완전한 데이터 보존

데이터 최적화:

  • 오래된 로그 데이터 정리
  • 검색 인덱스 최적화
  • 데이터베이스 성능 개선을 위한 정기 유지보수

배치 작업 모니터링 및 관리

1. Job Execution 모니터링

JobExecutionListener를 통한 배치 작업 추적:

실행 상태 추적:

  • 작업 시작/종료 시점 로깅 및 메트릭 수집
  • 실행 시간 측정을 통한 성능 모니터링
  • 성공/실패 통계를 통한 안정성 지표 관리

알림 시스템:

  • 배치 작업 실패 시 자동 알림 발송
  • 실행 시간 임계값 초과 시 성능 경고
  • 예외 정보 수집 및 로그 분석

2. 관리자 대시보드

REST API를 통한 배치 작업 관리:

실시간 상태 조회:

  • JobExplorer를 통한 등록된 Job 목록 조회
  • 각 Job의 최근 실행 이력 및 상태 확인
  • Step별 상세 실행 정보 제공

수동 제어 기능:

  • 관리자 권한으로 배치 Job 수동 실행
  • JobOperator를 통한 Job 실행 매개변수 설정
  • 실행 중인 Job 중단 및 재시작 기능

현재 구현 상태 (2023.12 기준):

  • module-batch 모듈 구조 구성 완료
  • Spring Batch 의존성 및 기본 설정 완료
  • 주기적 통계 집계를 위한 계획 수립 중

📊 구현 결과

배치 처리의 이점

Spring Batch 도입을 통해 다음과 같은 이점을 확보했습니다:

1. 안정성

  • 재시작 기능: 대용량 데이터 처리 중 오류 발생 시 중단된 지점부터 재시작
  • 트랜잭션 관리: 청크 단위 트랜잭션으로 부분 실패 시에도 처리된 데이터 보존
  • 오류 처리: Skip과 Retry 기능으로 일시적 오류에 대한 복원력 확보

2. 성능

  • 청크 기반 처리: 메모리 사용량 최적화로 100만 건 이상 데이터 안정적 처리
  • 병렬 처리: 멀티스레드 스텝으로 처리 속도 향상
  • 파티셔닝: 대용량 데이터를 여러 파티션으로 나누어 병렬 처리

3. 모니터링

  • 실행 이력: 모든 배치 작업의 실행 상태 및 결과 추적 가능
  • 메트릭 수집: 처리 시간, 처리 건수, 오류율 등 상세 메트릭 제공
  • 알림 기능: 작업 실패 시 자동 알림으로 신속한 대응 가능

4. 확장성

  • 수평 확장: 여러 서버에서 배치 작업 분산 실행 가능
  • 동적 설정: 파라미터를 통한 배치 작업 동적 설정 변경
  • 플러그인 구조: 새로운 배치 작업 추가 시 기존 인프라 재사용

기대 효과 및 향후 계획

Spring Batch 시스템 구축을 통해 다음과 같은 효과를 기대합니다:

처리 효율성:

  • 대용량 데이터 처리를 위한 청크 기반 메모리 관리
  • API 서버와 분리된 배치 처리로 시스템 안정성 향상
  • 자동화된 스케줄링으로 운영 부담 최소화

데이터 품질:

  • 체계적인 중복 제거 및 데이터 정제 작업
  • 일관된 통계 계산을 통한 신뢰할 수 있는 서비스 지표
  • 오류 처리 및 재시작 기능으로 데이터 손실 방지

확장성:

  • 새로운 배치 작업 추가 시 기존 인프라 재활용
  • 모듈화된 구조로 독립적인 배치 처리 확장
  • 멀티서버 환경에서의 분산 배치 처리 지원

이를 통해 API 서버의 성능에 영향을 주지 않으면서도 필요한 데이터 처리 작업을 안정적으로 수행할 수 있는 기반을 마련했습니다.
